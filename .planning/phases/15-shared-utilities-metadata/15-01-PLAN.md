  file: FileEntry;
  serverId: string;
  showHiddenFiles: boolean;

The hook must return this interface:
```typescript
interface UseFileContextMenuResult {
  // Context menu state
  contextMenu: { x: number; y: number } | null;
  handleContextMenu: (e: React.MouseEvent) => void;

  // Rename state
  isRenaming: boolean;
  renameValue: string;
  setRenameValue: (value: string) => void;
  handleRenameStart: () => void;
  handleRenameConfirm: () => Promise<void>;
  cancelRename: () => void;

  // File operation handlers (called from context menu buttons)
  handleDownload: () => Promise<void>;
  handleUpload: () => Promise<void>;
  handleDelete: () => Promise<void>;
  handleMoveTo: () => void;
  handleAddToFavorites: () => Promise<void>;
  handleUploadFolder: () => Promise<void>;
  handleDownloadFolder: () => Promise<void>;
}
```

Extract these items from FileItem.tsx into the hook:
1. `FolderUploadState` and `FolderDownloadState` interfaces (move to hook file)
2. `ConflictStrategy` type (move to hook file, or import from shared/types)
3. `formatBytes` function (keep it temporarily inside the hook file -- Plan 15-02 will replace with shared utility)
4. ALL useState calls: contextMenu, isRenaming, renameValue, activeOperationId, folderUploadState, folderDownloadState
5. ALL useRef calls: activeToastRef, folderUploadActiveRef, folderDownloadActiveRef, folderProgressCleanupRef, folderDownloadProgressCleanupRef

8. ALL useCallback handlers: handleContextMenu, handleCancelOperation, handleDownload, handleUpload, handleDelete, handleRenameStart, handleRenameConfirm, handleMoveTo, handleAddToFavorites, handleUploadFolder, handleRetryFailedFiles, handleDownloadFolder, handleRetryFailedDownloads

Critical implementation details:
- The hook must use `useCallback` for all handlers with correct dependency arrays to avoid stale closures
- `activeToastRef`, `folderUploadActiveRef`, `folderDownloadActiveRef` must be created inside the hook via `useRef` (NOT passed from component)
- `folderProgressCleanupRef` and `folderDownloadProgressCleanupRef` must be inside the hook, and the hook must include a cleanup effect that calls them on unmount
- `handleRetryFailedFiles` references `handleUploadFolder` and vice versa -- handle circular references with refs or careful useCallback ordering
- `handleRetryFailedDownloads` references `handleDownloadFolder` similarly
- The `setContextMenu(null)` calls at the start of each handler must remain (they close the menu when an action is selected)
- Import `toast` from 'sonner' in the hook file
- Import `FileEntry` from '../../shared/types'
- The hook returns a `cancelRename` function that simply calls `setIsRenaming(false)` (used by FileItem for Escape in rename input)

Follow the established hook pattern from `useFavorites.ts`: state + action callbacks returned as an object.
