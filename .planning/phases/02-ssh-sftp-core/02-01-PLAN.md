Create src/main/ssh/types.ts with comprehensive type definitions:

1. **Server** - Represents a server from SSH config or custom:
   - id: string (generated from host for SSH config, UUID for custom)
   - name: string (Host alias or display name)
   - host: string (HostName or IP)
   - port: number (default 22)
   - username: string
   - source: 'ssh-config' | 'custom'
   - keyPath?: string (IdentityFile path)
   - authMethod: 'key' | 'password' | 'agent'

2. **CustomConnection** - For user-added connections:
   - Extends Server without source (always 'custom')
   - displayName?: string

3. **ConnectionState** - State machine for connection lifecycle:
   ```typescript
   type ConnectionState =
     | { status: 'idle' }
     | { status: 'resolving' }
     | { status: 'authenticating' }
     | { status: 'loading-directory' }
     | { status: 'ready' }
     | { status: 'error'; message: string };
   ```

4. **FileEntry** - Directory listing entry:
   - name: string
   - path: string (full path)
   - size: number
   - modified: Date
   - isDirectory: boolean
   - isSymlink: boolean
   - permissions: string (e.g., '0755')
   - uid: number
   - gid: number
   - target?: string (symlink target if isSymlink)

5. **DirectoryListing** - Response from directory read:
   - path: string
   - entries: FileEntry[]

  <action>
Create src/main/ssh/config-parser.ts that parses ~/.ssh/config into Server objects.

Use the ssh-config library (see RESEARCH.md for patterns):

1. **parseSSHConfig(configContent: string): Server[]**
   - Parse raw SSH config content using SSHConfig.parse()
   - Filter entries where param === 'Host' and value !== '*'
   - For each host, use config.compute(hostName) to get merged settings
   - Map to Server type:
     - id: slugify host name (lowercase, replace spaces/special chars with hyphen)
     - name: host alias (the Host value)
     - host: computed.HostName or computed.Host (fallback to name)
     - port: computed.Port as number, default 22
     - username: computed.User or empty string
     - source: 'ssh-config'
     - keyPath: computed.IdentityFile (first one if array)
     - authMethod: 'key' if keyPath exists, else 'agent'

2. **getSSHConfigServers(): Server[]**
   - Read ~/.ssh/config from filesystem
   - Handle file not existing (return empty array)
   - Handle parse errors (log warning, return empty array)
   - Call parseSSHConfig with file content

Import path for home directory: `import { homedir } from 'os'`
Import for file reading: `import { readFileSync, existsSync } from 'fs'`

Handle edge cases from RESEARCH.md:
- SSH config uses first-match semantics - compute() handles this
- Host can be wildcard pattern - skip entries where Host is '*'
